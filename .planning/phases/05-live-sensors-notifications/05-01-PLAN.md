<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kernel sysfs -> core sensor facade | Sensor files can disappear or contain unparsable values; core returns `None` instead of raising. |
| Core sensor facade -> GTK UI | GUI must treat readings as untrusted optional values and avoid direct sysfs path assumptions. |
| GLib timer -> window lifecycle | Timer callbacks must not run against destroyed widgets. |
| GUI event -> desktop notification service | Notifications must not spam or leak through focused-window suppression rules. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of service | GTK live refresh | mitigate | Use `GLib.timeout_add_seconds(2, ...)` on the main loop; forbid `threading.Thread` and `GLib.idle_add` in Phase 5 modules. |
| T-05-02 | Spoofing / Integrity | Sensor values | mitigate | GUI uses only `read_sensors()` and renders `None` placeholders; no direct `/sys/class/hwmon` paths in GUI code. |
| T-05-03 | Denial of service | Timer lifecycle | mitigate | Store the source ID and call `GLib.source_remove()` on blocker routing and close/destroy. |
| T-05-04 | Safety / UX | Thermal display | mitigate | Temperature values are visible text plus bar state; color is not the only signal. |
| T-05-05 | Denial of service | Critical notifications | mitigate | Hysteresis sends only on threshold crossings and uses stable notification IDs. |
| T-05-06 | Privacy / UX | Focused notifications | mitigate | Focused windows use `Adw.Toast`; `Gio.Notification` is sent only while unfocused. |
</threat_model>
