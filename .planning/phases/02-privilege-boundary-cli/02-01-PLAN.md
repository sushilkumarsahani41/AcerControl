    stdout: str
    stderr: str

### Exit-code contract (privilege.py's primary cross-layer contract)

| Origin | Exit Code | Meaning | CLI-level response |
|--------|-----------|---------|---------------------|
| Wrapper | 0 | Success | Continue to read-back |
| Wrapper | 64 | EX_USAGE — argv allowlist reject | "internal error: wrapper rejected" → exit 2 |
| Wrapper | 71 | EX_OSERR — sysfs/file/systemctl IO failure | "Failed to write profile: …" → exit 1 |
| Wrapper | 77 | EX_NOPERM — wrapper called as non-root | "internal error: wrapper not privileged" → exit 1 |
| pkexec  | 126 | Auth dialog dismissed | "Authentication cancelled" → **exit 0** (PRIV-04 idempotent) |
| pkexec  | 127 | Not authorized / not granted / pkexec missing | "Authentication required but not granted" → exit 1 |
| run_privileged | 124 | local TimeoutExpired | "Operation timed out after 30s" → exit 1 |

## Phase 2 NEW: Wrapper argv contract (`libexec/*`)

```text
libexec/acercontrol-setprofile <kernel-value>
    kernel-value ∈ ("low-power", "quiet", "balanced", "balanced-performance", "performance")
    exit codes: 0 / 64 (EX_USAGE) / 71 (EX_OSERR) / 77 (EX_NOPERM)
    writes:     /sys/firmware/acpi/platform_profile  (open("w") + write)

libexec/acercontrol-set-boot-profile <kernel-value>
    kernel-value ∈ (same as above)
    exit codes: 0 / 64 / 71 / 77
    writes:     /etc/default/acercontrol  (atomic tempfile+rename; content `BOOT_PROFILE=<value>\n`)

libexec/acercontrol-manage-service <action> <service>
    action  ∈ ("enable", "disable", "start", "stop")
    service ∈ ("acer-performance.service",)            # OQ-01 carry-forward to Phase 6
    exit codes: 0 (forwarded from systemctl) / 64 / 71 / 77
    invokes:    subprocess.run(["systemctl", action, service], timeout=20)
```

**Defense-in-depth invariant (P2-NEW-01):** each wrapper hardcodes `ALLOWED_KERNEL_VALUES` (or `ALLOWED_ACTIONS` / `ALLOWED_SERVICES`) as a literal tuple. It MUST NOT do `from acercontrol.profiles import PROFILES` — `pkexec` scrubs `PYTHONPATH` (verified, pkexec(1) Noble), so the import would `ModuleNotFoundError` on every real-elevation invocation. The drift gate in Task 6 asserts `ALLOWED_KERNEL_VALUES == tuple(PROFILES.values())` at smoke time.

**Shebang invariant:** all three wrappers use `#!/usr/bin/python3` (absolute) per RESEARCH §Pattern 3 line 924. `pkexec` rebuilds PATH from a safe minimal environment; `/usr/bin/env python3` is not guaranteed resolvable.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| A: CLI ↔ polkit | User argv crosses into the polkit daemon. polkit selects an action (by `exec.path` match) and renders the auth dialog. From polkit's perspective, the CLI is **untrusted**. |
| B: polkit ↔ wrapper | After auth, `pkexec` invokes the wrapper as uid 0. The wrapper receives `argv[1:]` (CLI input, polkit-relayed). The wrapper is the **last line of defense** before the sysfs write. |
| C: CLI ↔ wrapper (sudo path) | When `$SSH_CONNECTION` is set, the CLI invokes `sudo -- <wrapper>` instead of `pkexec`. The wrapper is still the trust boundary; sudo provides the uid transition + auth UX. |
