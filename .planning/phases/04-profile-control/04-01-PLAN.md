<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GTK click -> profile state machine | Untrusted UI event requests a named profile; GUI must validate through `PROFILES` and not treat intent as state. |
| GUI -> privilege helper | Unprivileged GUI crosses into polkit/root wrapper through `run_privileged()` argv list. |
| Root wrapper -> kernel sysfs | `acercontrol-setprofile` writes `/sys/firmware/acpi/platform_profile`; wrapper allowlist remains the trust boundary. |
| Kernel/PPD read-back -> GUI | Actual state may differ from requested state; UI must render read-back truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering / Elevation of privilege | `ProfileControlPanel._on_profile_clicked` | mitigate | Use `PROFILES[requested_profile]` from fixed `ORDER` labels and pass `run_privileged(["acercontrol-setprofile", PROFILES[requested_profile]])`; no shell, no direct pkexec, no user-provided raw values. |
| T-04-02 | Elevation of privilege | `libexec/acercontrol-setprofile` direct local invocation | mitigate | Do not modify wrapper; rely on existing root check and `ALLOWED_KERNEL_VALUES`; smoke verifies GUI calls the wrapper by name through `run_privileged()` rather than duplicating privileged writes. |
| T-04-03 | Spoofing / Repudiation | Active profile highlight | mitigate | Active styling comes only from `read_profile()` initial state or 250 ms read-back; pending click disables controls but does not move highlight. |
| T-04-04 | Information disclosure / Safety | GUI labels and status text | mitigate | Use user-facing profile labels and `Profile.display`; smoke forbids raw kernel literals `low-power` and `balanced-performance` in GUI files and preserves diagnostics-only carve-out. |
| T-04-05 | Denial of service / UX safety | Polkit cancellation path | mitigate | `PrivilegedResult.cancelled` clears pending, re-enables controls, preserves actual highlight, shows `Authorization cancelled` for 3 seconds, and does not read back or force PPD banner. |
</threat_model>
